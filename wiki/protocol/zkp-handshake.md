Privacy-preserving mutual authentication between a human and an AI agent using zero-knowledge proofs. Both parties independently prove identity claims without revealing anything beyond policy satisfaction, and both proofs are verified atomically in a single on-chain transaction.

## Overview

The mutual handshake is the core authentication primitive in Bolyra. A human proves membership in an identity group (Merkle tree inclusion) while an agent proves it holds a valid, non-expired, operator-signed credential with sufficient permissions. Neither party learns anything about the other beyond the fact that verification passed.

Both proofs are bound to the same session nonce to prevent mix-and-match attacks. The on-chain registry verifies both atomically and records the agent's scope commitment as the seed for any subsequent delegation chain.

## Key Concepts

**Session Nonce** -- A cryptographically random value (>= 128 bits) generated by the verifier. Binds both proofs to a single session. Checked for freshness against an on-chain used-nonce mapping. The SDK encodes a Unix timestamp in the upper bits for freshness checking:

```ts
// Layout: nonce = (unix_seconds << 64) | random_64_bits
export function defaultNonce(): bigint {
  const ts = BigInt(Math.floor(Date.now() / 1000));
  const rand = BigInt('0x' + randomBytes(8).toString('hex'));
  return (ts << 64n) | rand;
}
```

**Identity Commitment (Human)** -- `Poseidon2(Ax, Ay)` where `(Ax, Ay)` is the Baby Jubjub public key derived from the secret via `BabyPbk()`. Compatible with Semaphore v4.

**Credential Commitment (Agent)** -- `Poseidon5(modelHash, Ax, Ay, permissionBitmask, expiryTimestamp)`. The operator signs this commitment with EdDSA.

**Nullifier** -- Deterministic per identity per scope. Human: `Poseidon2(scope, secret)`. Agent: `Poseidon2(credentialCommitment, sessionNonce)`.

**Nonce Binding** -- `Poseidon2(nullifierHash, sessionNonce)`. Ties the human proof to the specific session.

**Scope Commitment** -- `Poseidon3(permissionBitmask, credentialCommitment, expiryTimestamp)`. Published as the agent's public output. Serves as the chain seed for delegation.

## How It Works

### Protocol Flow

1. **Verifier generates nonce** -- Fresh, cryptographically random, at least 128 bits.

2. **Human generates Groth16 proof** (HumanUniqueness circuit):
   - Proves identity commitment is a leaf in the human Merkle tree
   - Outputs scope-bound nullifier for Sybil detection
   - Outputs nonce binding tying proof to the session

3. **Agent generates Groth16 proof** (AgentPolicy circuit) concurrently:
   - Proves credential commitment is a leaf in the agent Merkle tree
   - Verifies operator EdDSA signature over credential commitment
   - Checks permission bitmask satisfies required scope policy
   - Checks credential has not expired
   - Outputs identity-bound scope commitment for delegation chain entry

4. **Both proofs submitted atomically** to on-chain registry in a single transaction.

5. **Registry verifies** (in order):
   - Session nonce freshness (mark used atomically)
   - Nonce equality across both proofs and transaction argument
   - Human nullifier not revoked
   - Both Merkle roots valid against 30-entry root history buffers
   - Both Groth16 proofs valid

6. **On success**: nonce recorded, agent scope commitment stored as delegation chain seed, `HandshakeVerified` event emitted.

### Public Signal Layouts

Human (5 signals): `[humanMerkleRoot, nullifierHash, nonceBinding, scope, sessionNonce]`

Agent (6 signals): `[agentMerkleRoot, nullifierHash, scopeCommitment, requiredScopeMask, currentTimestamp, sessionNonce]`

### SDK API

```ts
// Generate both proofs in parallel
const { humanProof, agentProof, nonce } = await proveHandshake(
  humanIdentity,
  agentCredential,
  { scope: 1n }
);

// Off-chain verify (for testing / gating)
const result = await verifyHandshake(humanProof, agentProof, nonce);
// result.verified, result.humanNullifier, result.agentNullifier, result.scopeCommitment

// On-chain: submit to IdentityRegistry.verifyHandshake()
```

The SDK validates nonce binding before returning `verified: true` -- if the nonce in the proof's public signals does not match the expected nonce, verification returns `verified: false` without throwing.

## Current Status

- Phase 1 (Proof of Enrollment) is live. Sybil resistance is a stub; Phase 4 replaces it with behavioral/cognitive enrollment.
- HumanUniqueness uses Groth16 only (reuses Semaphore v4 ceremony at depth 20).
- AgentPolicy supports both Groth16 and PLONK, but Groth16 is required by the spec.
- Conformance suite: 10 handshake vectors, 2 signature vectors, 5 Merkle vectors.
