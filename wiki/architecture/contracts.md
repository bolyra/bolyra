Four Solidity contracts implement on-chain ZKP verification and identity management for the Bolyra protocol. Three are auto-generated Groth16 verifiers; one is the hand-written `IdentityRegistry` that orchestrates enrollment, handshake verification, delegation, and revocation.

## Overview

The contracts are deployed on **Base Sepolia** (chain ID 84532). They verify Groth16 proofs generated by the Circom circuits (HumanUniqueness, AgentPolicy, Delegation) and maintain two Lean Incremental Merkle Trees (human and agent) on-chain.

## Key Concepts

- **Groth16 only on-chain.** While the circuits support both Groth16 and PLONK, only Groth16 verifiers are deployed -- roughly 10x cheaper gas than PLONK verification.
- **Auto-generated verifiers.** `HumanVerifier.sol`, `AgentVerifier.sol`, and `DelegationVerifier.sol` are produced by `snarkjs zkey export solidityverifier`. They contain hardcoded verification key constants and must be regenerated whenever a circuit's `.zkey` changes.
- **LeanIMT.** Both identity trees use `@zk-kit/lean-imt.sol` (the Semaphore v4 Lean Incremental Merkle Tree). Requires `PoseidonT3` library deployment.
- **Root history buffer.** Both human and agent trees maintain a circular buffer of the last 30 roots. This prevents valid in-flight proofs from failing when new enrollments update the tree root during proof generation.

## How It Works

### Contract Architecture

```
PoseidonT3 (library)
    |
    v
IdentityRegistry ----> HumanVerifier (Groth16Verifier)
    |               --> AgentVerifier (AgentGroth16Verifier)
    |               --> DelegationVerifier (DelegationGroth16Verifier)
    |
    +-- humanTree (LeanIMT)
    +-- agentTree (LeanIMT)
```

### Contracts

| Contract | File | Type | Public Signals |
|---|---|---|---|
| `Groth16Verifier` | `HumanVerifier.sol` | Auto-generated | 5: merkleRoot, nullifier, nonceBinding, scope, sessionNonce |
| `AgentGroth16Verifier` | `AgentVerifier.sol` | Auto-generated | 6: merkleRoot, nullifier, scopeCommitment, requiredScope, currentTimestamp, sessionNonce |
| `DelegationGroth16Verifier` | `DelegationVerifier.sol` | Auto-generated | 6: newScopeCommitment, delegationNullifier, delegateeMerkleRoot, previousScopeCommitment, sessionNonce, currentTimestamp |
| `IdentityRegistry` | `IdentityRegistry.sol` | Hand-written | N/A (orchestrator) |

### IdentityRegistry Functions

**Enrollment (owner-only):**
- `enrollHuman(identityCommitment)` -- insert into humanTree
- `enrollAgent(credentialCommitment)` -- insert into agentTree
- `enrollHumanBatch(...)` / `enrollAgentBatch(...)` -- batch variants

**Verification:**
- `verifyHandshake(humanProof, humanPubSignals, agentProof, agentPubSignals, sessionNonce)` -- verifies a mutual human+agent handshake in a single transaction. Checks nonce freshness, revocation status, Merkle root validity (30-root buffer), both Groth16 proofs, agent timestamp freshness (5-minute skew), and seeds the delegation chain state.
- `verifyDelegation(proof, pubSignals, sessionNonce)` -- verifies a single delegation hop. Requires a prior handshake for the same nonce. Enforces chain-linking from on-chain state (not caller-supplied), delegation nullifier replay protection, max 3 hops per session, delegatee Merkle root validity, and timestamp freshness.

**Revocation:**
- `revokeHuman(nullifier)` -- marks a nullifier as revoked (checked during handshake)
- `revokeAgent(oldCredentialCommitment, siblingNodes)` -- zeros the leaf in agentTree via LeanIMT update, invalidating all subsequent proofs

**Security hardening applied:**
- Nonce-in-proof equality enforcement (both proofs must embed the same sessionNonce)
- Public signal length pinning (HUMAN_PUBSIG_LEN=5, AGENT_PUBSIG_LEN=6, DELEGATION_PUBSIG_LEN=6) with layout version constants
- Timestamp freshness (5-minute skew for both agent and delegation proofs)
- On-chain delegation chain state (eliminates caller-supplied `expectedPreviousScopeCommitment`)
- Max delegation depth of 3 hops per session

### Deployment

Deploy script (`contracts/scripts/deploy.js`) deploys 5 contracts in order:

1. `PoseidonT3` library
2. `Groth16Verifier` (HumanUniqueness)
3. `AgentGroth16Verifier` (AgentPolicy)
4. `DelegationGroth16Verifier` (Delegation)
5. `IdentityRegistry` (linked to PoseidonT3, constructed with the three verifier addresses)

Deployment artifacts are saved to `contracts/deployments/base-sepolia.json`.

```bash
# Local (Hardhat network):
cd contracts && npm run deploy:local

# Base Sepolia:
cd contracts && npm run deploy:base-sepolia
# Requires BASE_SEPOLIA_RPC_URL and DEPLOYER_PRIVATE_KEY env vars
```
