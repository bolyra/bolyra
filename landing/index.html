Every call, checked

Four checks on every tool call.

1

Credential Verification

verifyBundle() checks ZKP proof, delegation chain, scope commitment, expiry. 401 on failure.

2

Tool Policy

YAML config maps tool names to permission bitmasks. read_file needs READ_DATA, delete needs WRITE_DATA. 403 on insufficient.

3

Replay Protection

Session nonce checked against MemoryNonceStore or Redis. Same nonce twice = rejected.

4

Audit Receipt

ES256K-signed receipt for every decision. Allow or deny. Timestamped.

Packages

Integrations and tools

Plug Bolyra into your agent framework, wallet, or payment flow.

MCP

MCP Middleware

v0.6.5: Dev mode — authenticated MCP server in 60 seconds, zero circuit artifacts. withBolyraAuthStdio(server, { devMode: true, toolPolicy, receiptSigner }). Per-tool permission gating, delegation chain support, signed verification receipts, stdio + HTTP transports. Listed on the official MCP Registry. Runnable example included. npm install @bolyra/mcp@0.6.5.

GW

MCP Auth Gateway

v0.6.0: Standalone reverse proxy that sits in front of any MCP server. Verifies agent credentials, enforces per-tool permission policies, prevents replay, and emits ES256K-signed receipts for every allow and deny, hash-chained so deleted or reordered log lines are detectable. Credential binding: register credentials in gateway.yaml (or --credentials) and Core mode fail-closes on unknown commitments, forged masks, or expired registrations; production mode compiles the same registry, with expiryTimestamp per entry, into resolveCredential for scopeCommitment binding. RedisNonceStore for multi-instance replay protection. New: serves /.well-known/bolyra-signers.json (Receipt Signer Discovery v1) so third parties can fetch the receipt-signer trust anchor instead of exchanging keys manually. Listed on the official MCP Registry. Docker: docker run ghcr.io/bolyra/gateway --target <your-mcp>. Also: npx @bolyra/gateway or createGatewayMiddleware for Express/Hono/Fastify. npm install @bolyra/gateway@0.6.0.

SH

MCP Shield

v0.1.0: Stdio MCP auth proxy. Wrap any stdio MCP server with per-tool permission enforcement, replay protection, and audit receipts. No code changes to the server. npx @bolyra/shield --server "npx some-mcp-server" --dev. Shield covers stdio; Gateway covers HTTP. Both use the same verifyBundle() and checkToolPolicy(). Listed on the official MCP Registry. npm install @bolyra/shield.

R

Signed Receipts

v0.9.0: Auth + commerce receipts. Every verification and payment authorization decision produces a cryptographically signed receipt. secp256k1 signatures, keccak256 hashing, EVM-compatible. Hash-chained logs: chain fields are signed inside each receipt, so edits, deletions, insertions, and reordering break verification. bolyra-receipt-verify CLI included. New: Receipt Signer Discovery v1 parser (parseSignerDiscovery) for consuming /.well-known/bolyra-signers.json trust anchors. Try both sides: the public receipt test corpus (producer side — independently verifiable, no contact with us needed) and the scoring consumer reference (what an indexer or risk engine does with verified receipts: verify first, fail closed, then extract per-actor features). npm install @bolyra/receipts@0.9.0.

$

Payment Protocols

v0.7.0: Stripe ACP spend gating. verifyStripeACPSpend(ctx, amount, currency, 'authorize' | 'confirm') gates PaymentIntents against delegated scope. confirm fails closed without SIGN_ON_BEHALF. Signed commerce receipts on every authorization. New: runnable e2e demo -- $25 allowed, $480 rejected, confirm fails closed, 3/3 receipts verified. npm install @bolyra/payment-protocols@0.7.0.

OC

OpenClaw Adapter

ZKP-native trust verification for OpenClaw's onAgentVerify hook. 5-dimension scoring: proof validity, expiry, permissions, freshness, scope. npm install @bolyra/openclaw@0.3.2

AI

Vercel AI SDK

Works with generateText(), streamText(), useChat(). npm install @bolyra/ai.

OA

OpenAI Agents SDK

v0.1.0: Auth guardrail + tool wrapper for the OpenAI Agents SDK. BolyraAuthGuardrail gates agent runs. BolyraToolWrapper enforces per-tool permissions. bolyra_mcp_auth() wraps MCP server connections with auth headers. SD-JWT and gateway modes. Tracing integration. pip install bolyra-agents.

CA

CrewAI Adapter

v0.2.0: Three BaseTool subclasses for CrewAI multi-agent workflows. BolyraAuthTool, BolyraDelegateTool, BolyraSDJWTTool. Pre-execution BolyraGuard with guard_tools() wrapping. Thread-safe BolyraSession chains handshake, delegation, and SD-JWT flows. BolyraDelegationFlow for per-agent scoping with proof envelope audit trail. 99 tests. pip install bolyra-crewai.

LC

LangChain Adapter

v0.1.0: Three proper BaseTool subclasses for LangChain agents. BolyraAuthTool (ZKP mutual handshake), BolyraDelegateTool (ZKP delegation), BolyraSDJWTTool (pure-Python SD-JWT delegation, no Node.js required). Session manager for auth-then-delegate chains. Dev mode with auto-generated credentials. pip install bolyra-langchain.

PY

Python SDK

v0.5.0: Types, validation, and subprocess bridge to the Node SDK for ZKP proving. TS SDK at v0.6.1 (snarkjs now lazy-loaded — Bolyra Core paths never import it). New: pure-Python SD-JWT delegation module (allow(), present(), verify()) matching the TypeScript wire format exactly. Ed25519 + PyJWT. pip install bolyra.

CLI

Credential CLI

v0.6.0: bolyra verify — a spawnable external verifier any MCP host can call: one JSON request on stdin, one allow/deny verdict on stdout, fail-closed on everything else. Implements the open External Verifier Contract v1 with conformance vectors. Plus: bolyra shield test — safety test runner for MCP servers. bolyra run — stdio→HTTP with auth. bolyra observe — live viewer + policy gen. bolyra replay — receipt re-evaluation. bolyra doctor — health check. Also: cred, key, receipt verify, receipt verify-chain (whole-log tamper evidence, now with --signer-from <url> signer discovery), dev from-receipt. npm install -g @bolyra/cli@0.6.0.

CI

GitHub Action

v0.1.0: Bolyra Replay Check — CI for agent behavior regressions. Replays receipt history against your policy on every PR. Posts a comment showing regressions (allow→deny) and relaxations (deny→allow). Fails the check if regressions are found. Drop-in: uses: bolyra/bolyra/actions/replay-check@main. View on GitHub.

DW

Delegation Wallet

Live: Human-facing consent screen for AI agent permissions. Create delegations, pick tools, set spend limits, choose expiry. OAuth-style approval flow. Receipt timeline for every agent action. Revoke live. app.bolyra.ai.

ZK

Differentiation

Why not just OAuth?

Capability OAuth / JWT Bolyra
Delegated scope narrowing Manual policy engine Circuit-enforced, one-way

Standards

Built on open standards

Not just code — a protocol with formal specifications, a DID method, and conformance tests.

IETF

Draft (preparing for submission)

draft-bolyra-mutual-zkp-auth-01

Formal protocol specification for mutual zero-knowledge proof authentication. Defines message flows, proof formats, and verification procedures. SDK (v0.6.x) aligns all circuits to Groth16 (REQUIRED), PLONK (OPTIONAL). Canonical 6-element Delegation public-signals layout locked for IETF submission. External Verifier Contract v1 is proof-system-agnostic (kind: classical | zk | external).

W3C

DID Method

did:bolyra

W3C Decentralized Identifier method for Bolyra identities. Supports both human and agent DID documents with ZKP-aware verification methods.

Conformance

Test Vectors

67 vectors, all passing

Conformance suite covers handshake, 2-hop delegation chain, forged-token EdDSA, nullifier-per-nonce, Poseidon3/4 binding sensitivity, LeanIMT root edges, canonical 6-element public-signals layout, financial scope narrowing, and cumulative-invariant rejection.
