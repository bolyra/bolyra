Landing Page Redesign: New Hero + Page Flow

Based on Codex's review: lead with the gateway product, not the protocol. Show before/after. Move ZK deep-dive below the fold.

Current Hero (5/10)

bolyra.ai — current

Bolyra

Verifiable delegated authority for AI agents. The human signs a scoped receipt; middleware verifies it before the agent calls a tool.

SD-JWT receipts + KB-JWT proof-of-possession + IETF status-list revocation. Per-call audience, action, permission, amount-cap, and expiry checks. Apache-2.0, one dep, 95 conformance tests. ZKP cohort v0.7.0 shipped...

npm @bolyra/sdk npm @bolyra/delegation GitHub

npm install @bolyra/sdk @bolyra/mcp @bolyra/gateway @bolyra/receipts @bolyra/payment-protocols

Problems

  • "Verifiable delegated authority" is abstract
  • Floods with acronyms (SD-JWT, KB-JWT, IETF...)
  • 5 npm packages at once, unclear where to start
  • Reads like release notes

Proposed Hero

bolyra.ai — proposed

Auth for AI agents

Your MCP server has no idea
who's calling.

Put Bolyra in front of any MCP server. It verifies agent credentials, enforces per-tool permissions, blocks replay, and signs audit receipts.

Protect an MCP server View GitHub

npx @bolyra/gateway --target http://localhost:3000/mcp

Improvements

  • Problem-first headline: "has no idea who's calling"
  • One clear CTA: "Protect an MCP server"
  • One install command (gateway), not five packages
  • Outcome-focused subcopy, zero acronyms
